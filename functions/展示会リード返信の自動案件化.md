# 展示会リード：返信の自動案件化（watchLeadReplies）

リストメールの **Cc に指定した検知用メールボックス** を約15分ごとに読み、
**リードのメールアドレスからの返信** を見つけたら、そのリードを **自動で案件化** します
（案件＋商談記録を作成し、リードを「案件化済」に）。手動の「返信あり→案件化」も従来どおり使えます。

## 仕組み

- スケジュール関数 `watchLeadReplies`（15分ごと）
- 各会社の「自社情報マスタ → リード返信検知用Cc」のメールボックスを Gmail API で読む
- 直近2日の受信から差出人メールを集め、未案件化リードの `email` と一致したら案件化
- 二重処理防止：処理済みメールIDを記録＋リードを「案件化済」にする

## 必要な設定（3つ）

### 1. サービスアカウントに Gmail 読み取り権限を追加

送信用に使っている既存のサービスアカウント（`GMAIL_SA_KEY`）に、
**ドメイン全体の委任スコープ** を追加します。

Google Workspace 管理コンソール → セキュリティ → アクセスとデータ管理 → **API の制御**
→ **ドメイン全体の委任** → 既存のサービスアカウント（クライアントID）を編集し、
スコープに次を **追加**（既存の `gmail.send` は残す）:

```
https://www.googleapis.com/auth/gmail.readonly
```

複数スコープはカンマ区切りで併記します:
```
https://www.googleapis.com/auth/gmail.send,https://www.googleapis.com/auth/gmail.readonly
```

### 2. 検知用Cc は「実在するユーザーのメールボックス」にする

委任は実在ユーザーのメールボックスを読む仕組みのため、検知用Cc は
**Google Workspace の実在アカウント**（例 `lead-watch@n-airmore.com` を実ユーザーとして作成）
にしてください。**グループ／エイリアス宛は読めません**。

商談管理 → マスタ・設定 → 自社情報(見積書) → 「リードメール返信検知用Cc」に設定します（会社ごと）。

### 3. デプロイ

```
cd C:\Users\saraz\Desktop\airmore-task
firebase deploy --only functions
```

`watchLeadReplies` が追加されます（既存の関数はそのまま）。

## 確認方法

1. テスト用に、あるリードの `email` を自分の別アドレスにして「送信済」にする
2. その別アドレスから検知用Cc宛（またはCcに検知用アドレスを含めて）返信する
3. 15分以内に、そのリードが「案件化済」になり、案件ボードに案件が現れれば成功

うまくいかない場合は Firebase コンソール → Functions → `watchLeadReplies` のログを確認してください。
`gmail read failed`（委任スコープ未設定 or Cc が実在ユーザーでない）などの原因が出ます。

## 費用・注意

- 15分ごとに、検知用Ccを設定した会社の数だけ Gmail を読みます（未設定の会社はスキップ）
- 一致は「リードの email」と「返信差出人」の完全一致（大文字小文字は無視）
- 同じ返信で二重に案件化されることはありません（処理済みメールID＋リードの案件化済で防止）
