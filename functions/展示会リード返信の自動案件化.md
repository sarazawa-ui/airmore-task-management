# 展示会リード：返信の自動案件化（watchLeadReplies）

リストメールを送った後、**担当メンバー各自のメールボックス**を約15分ごとに確認し、
**リードのメールアドレスからの返信** を見つけたら、そのリードを **自動で案件化** します
（案件＋商談記録を作成し、リードを「案件化済」に）。手動の「返信あり→案件化」も従来どおり使えます。

**専用の検知用メールアドレスは不要です。** リードは返信を「送信した担当者本人」に返すため、
各メンバーの受信を見れば返信を拾えます。

## 仕組み

- スケジュール関数 `watchLeadReplies`（15分ごと）
- 各会社の「メンバー各自のGoogleアカウント」のメールボックスを Gmail API で読む
- 未案件化リードのメールアドレスに絞って検索（`from:(...)`）するので、拾うのは実際のリード返信だけ
- 一致したら案件化。二重案件化はリードの「案件化済」で防止
- 自社情報マスタの「検知用Cc」を設定していれば、そのボックスも追加で確認します（任意）

## 必要な設定（2つ）

### 1. サービスアカウントに Gmail 読み取り権限を追加

送信用に使っている既存のサービスアカウント（`GMAIL_SA_KEY`）に、
**ドメイン全体の委任スコープ** を追加します。

Google Workspace 管理コンソール → セキュリティ → アクセスとデータ管理 → **API の制御**
→ **ドメイン全体の委任** → 既存のサービスアカウント（クライアントID）を編集し、
スコープに次を **追加**（既存の `gmail.send` は残す）:

```
https://www.googleapis.com/auth/gmail.readonly
```

複数スコープはカンマ区切りで併記します:
```
https://www.googleapis.com/auth/gmail.send,https://www.googleapis.com/auth/gmail.readonly
```

これで、そのドメインの **各ユーザーのメールボックス** を読めるようになります。

> **メンバーが複数のドメインにまたがる場合**（例: `leaklab-japan.com` と `n-airmore.com`）
> は、**各ドメインの管理コンソールで同じ委任を登録**してください。
> 未登録ドメインのユーザーのメールボックスは読めず、そのユーザー宛の返信は検知されません
> （ログに `read failed for mailbox ...` と出ます）。委任済みの他ドメインで拾えれば問題ありません。

## 前提

- 各メンバーの **Googleアカウント（メール）** が Hittatsu に登録されていること
  （設定 → 個人設定でログインすると自動登録されます）。このメールのメールボックスを確認します。
- リード側の **メールアドレス** が登録されていること（返信の差出人と照合するため）。
- 返信が担当者本人（メンバー）宛に届くこと。転送などで別アドレスにしか届かない場合は、
  そのアドレスを検知用Cc（自社情報マスタ）に設定してください。

## 確認方法

1. あるリードの `email` を自分の別アドレスにして「送信済」にする
2. その別アドレスから、担当メンバー本人宛にメールを送る（リード→担当者への返信を再現）
3. 15分以内に、そのリードが「案件化済」になり、案件ボードに案件が現れれば成功

うまくいかない場合は Firebase コンソール → Functions → `watchLeadReplies` のログを確認してください。
`read failed for mailbox ...`（委任スコープ未設定 or そのドメインが未委任）などの原因が出ます。

## 費用・注意

- 15分ごとに、未案件化リードがある会社の「メンバー数分」だけ Gmail を検索します
  （`from:(リードのメール)` で絞るため、各検索が返す件数はごく少数）
- 一致は「リードの email」と「返信差出人」の完全一致（大文字小文字は無視）
- 同じ返信で二重に案件化されることはありません（リードの案件化済で除外）
