🏢 ワークスペース設定

ヘッダーのワークスペース切替メニューや「マイタスク横断」ビューに表示される識別名です。参加メンバー全員に共有されます。

⚠ 危険な操作（オーナー専用）

ワークスペースを削除すると、クラウド上のすべてのデータ（プロジェクト・タスク・履歴）が失われ、参加メンバー全員がアクセスできなくなります。この操作は取り消せません。

🏛 会社名（プロフィール）

💡 会社名・ロゴは このブラウザのみ に保存され、どのワークスペースに切り替えても共通で表示されます。他メンバーには共有されません。
ロゴなし
推奨：正方形 256×256px 以上 / PNG・JPG・SVG（5MB以下）
ヘッダーやブラウザタブのタイトル、通知メールの署名に使用されます。

🚪 ログアウト

ログアウトすると、このブラウザのセッションが終了します。
ローカルに保存されたデータは保持され、再ログイン後に復元されます。クラウド同期中のワークスペースのデータも安全です。

👤 個人設定（このアカウントのみ）

💡 ここで設定する内容は あなたのGoogleアカウント でログインしたときのみ反映されます。
他のチームメンバーには影響しません。同じワークスペース内でも各自が独立した設定を持てます。
週間カレンダーの先頭の曜日（個人設定）。
あなたが1日に作業に充てられる最大時間。8時間 = 480分。
メール対応・突発対応・休憩など、計画外の雑務に確保する時間。自分のタスク + Googleカレンダー予定の合計が「実働可能 − 雑務対応時間」を超える場合に警告します。
チェックON：超過する場合はカレンダー上にドロップできません。OFF：警告のみ表示し配置は許可。
タスクのチェックボックスをONにした際、登録メンバー（自分以外）の Gmail アドレス宛にGmail経由で完了通知を送信します。
※ Gmail送信権限が必要です。権限が無い場合、初回送信時に再ログインを促します。
※ 自動送信ではなく、毎回確認ダイアログが出ます。
本日のタスクで「時刻」が設定されているものについて、開始10分前に担当者のGoogleアカウントへリマインドメールを自動送信します。
※ ブラウザを開いている間のみ動作します（バックグラウンド送信にはサーバ実装が必要）。
※ 担当者のGoogleアカウントがメンバー一覧に登録されている必要があります。
※ 同じタスクは1日1回までしか送信されません。

      追加メンバー（ゲスト）

        ※ Googleでログインしたメンバーは「メンバー・招待」タブで自動管理。ここの「追加メンバー」はGoogleアカウントを持たないゲスト枠として使えます。

        ☁️ クラウド同期（Firebase Firestore）

        リアルタイムでチームメンバー間でデータを同期します。
        Firebase Console でプロジェクトを作成し、Firestore Database 有効化、Authentication でGoogleログインを有効化、Webアプリ追加で取得した設定をここに貼り付けてください。

        ○ 未設定
        通常はデフォルト設定のままで動作します。別のFirebaseプロジェクトに接続したい場合のみ上書きしてください。
        Firebase Console → プロジェクト設定 → 全般 → 「マイアプリ」 → ウェブアプリの設定 → SDK 設定と構成 → 「構成」をコピー

        👥 参加メンバー

        ※ 各メンバーがGoogleでログインすると、ここに自動追加されます（最初にログインした方がオーナー）。

        ⚙ Firestore セキュリティルール（必須）：
        Firebase Console → Firestore Database → ルールタブ で下記を貼り付けて「公開」してください。
        rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /workspaces/{wsId} {
      // 認証済みユーザーなら誰でも読み書き可
      // (招待URLの秘匿性とFirebase Auth認証が実質的なアクセス制御)
      allow read, write: if request.auth != null;
    }
  }
}
        ※ より厳密な「メンバーのみ」ルールは招待URL経由の参加時に問題が起きるため、シンプルなこのルールを推奨します。招待URLを知っている＋Googleログイン済みの人だけがアクセス可能です。

        🔐 Authentication 設定：
        Firebase Console → Build → Authentication → Sign-in method → 「Google」を有効化
        Settings → Authorized domains に を追加
